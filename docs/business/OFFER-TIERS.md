# Voila! Offer Tiers

Status: draft  
Milestone: v0.2.3-public-beta-commercial-positioning

Voila! is currently in public beta. The final licensing and commercial model are still under evaluation.

This document defines the intended offer tiers without changing the license status of the repository.

---

## 1. Community

For testers, learners, early users, and people evaluating Voila! locally.

Includes:

- public beta access
- local-first usage
- basic PDF/manual learning workflow
- release notes
- checksum files
- public documentation
- best-effort feedback review

Does not include:

- commercial usage license
- guaranteed support
- SLA
- redistribution rights
- white-label rights
- paid support
- final open-source license commitment

Positioning:

Voila! Community is for evaluation, testing, feedback, and early validation.

---

## 2. Supporter

For people who want to support development.

Possible benefits:

- supporter updates
- early roadmap notes
- priority feedback review
- supporter mention, if requested
- access to supporter package when available

Important:

Supporter payments support development. They do not automatically grant a commercial license unless explicitly stated in a future paid license.

Possible platforms:

- GitHub Sponsors
- Gumroad
- Lemon Squeezy
- Buy Me a Coffee
- Ko-fi

---

## 3. Pro

For individual power users.

Potential future benefits:

- polished Windows installer
- easier updates
- larger processing limits
- advanced export options
- saved projects
- custom glossary
- advanced OCR/language workflow
- packaged offline runtime
- priority packaging fixes

Possible pricing ideas:

- one-time paid download
- annual Pro license
- early-access supporter price
- discounted beta license

Important:

Pro should not be implemented until the value is clear and support expectations are realistic.

---

## 4. Business

For companies, trainers, documentation teams, and internal knowledge workflows.

Possible offer:

- commercial usage license
- custom onboarding
- custom language packs
- private deployment guidance
- invoice-based payment
- priority support
- private build
- maintenance agreement

Possible pricing ideas:

- custom quote
- annual business license
- per-company license
- paid setup
- paid support package

Important:

Business terms should be handled separately from public beta documentation.

---

## Current recommendation

For the next public step:

1. Keep Community public beta available.
2. Add Supporter option first.
3. Prepare Pro as a future paid package.
4. Offer Business only by direct inquiry.
5. Do not add a final LICENSE yet.
6. Do not add license key enforcement yet.
