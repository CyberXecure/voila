# Voila! Feedback Strategy

## Goal

Collect practical early-user feedback before adding large new features.

## Feedback priorities

### 1. Clarity

Questions:

- Do you understand what Voila! does in the first 30 seconds?
- Is the README clear?
- Is the release structure clear?
- Is the language-pack RC1 explanation clear?

### 2. Utility

Questions:

- Would you use PDF-to-course generation?
- Which generated outputs are most useful?
  - outline
  - course markdown
  - glossary
  - quiz
  - flashcards
  - OCR review
- What type of PDFs would you test with?

### 3. Language packs

Questions:

- Are English and Romanian enough for the first RC?
- Which future language matters most?
  - German
  - Spanish
  - Italian
  - Portuguese
- Would you help review translations?

### 4. Monetization

Questions:

- Would you support the project financially?
- What would make a supporter package valuable?
- Would you prefer:
  - one-time supporter package
  - monthly sponsorship
  - paid Pro package later
  - commercial license later

## Recommended channels

Use simple channels first:

- direct messages to 5-10 testers
- GitHub Issues for technical feedback
- a simple form for non-technical feedback
- LinkedIn post for public visibility

## What not to collect yet

Do not collect large feature requests too early.

Avoid:

- enterprise feature planning
- multi-user accounts
- cloud sync
- complex payment flows
- forced language expansion
- licensing debates before user interest is validated
