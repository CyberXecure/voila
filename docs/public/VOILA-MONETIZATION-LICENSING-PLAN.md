# Voila! Monetization and Licensing Plan

## Current status

Voila! is publicly visible on GitHub, but no LICENSE file has been added yet.

This is intentional.

The commercial model is still under evaluation, and the project should not grant broad reuse rights until the model is decided.

## Near-term monetization direction

Recommended next step:

- public beta feedback
- supporter package validation
- no paid lock-in yet
- no major feature development before validation

## Possible monetization paths

### 1. Supporter package

Best first option.

Good for:

- early supporters
- people who want the project to continue
- users who want easier builds, docs, examples, and priority feedback

### 2. Paid Pro package later

Possible later.

Good for:

- polished installer
- extra templates
- more language packs
- better exports
- batch processing
- priority support

### 3. Commercial license later

Possible later.

Good for:

- companies
- schools
- training providers
- internal use cases

## Licensing options to evaluate later

Do not decide yet.

Possible future models:

- proprietary / source-available
- dual license
- open-core
- permissive open-source plus paid services
- commercial license for business use

## What not to do now

Avoid:

- adding LICENSE before the model is clear
- promising open-source rights accidentally
- adding payment logic
- building cloud accounts
- building Pro features before validating demand
- forcing all planned languages into the first RC
