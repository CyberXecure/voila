# v0.7.4-owner-local-ocr-math-report-smoke-fixture-no-build-no-distribution

## Scope

This owner-local milestone adds a local smoke fixture and validation check for the OCR Math diagnostic report UI flow completed in v0.7.1-v0.7.3.

The check creates a temporary local fixture outside the repository with:

- `ocr_math_report.md`
- `ocr_math_report.json`
- a stable local course/document id: `v0-7-4-ocr-math-smoke-course`

It validates the existing owner-local routes:

- `/owner/ocr-math-report/{course_id}/summary.json`
- `/owner/ocr-math-report/{course_id}/ocr_math_report.md`
- `/owner/ocr-math-report/{course_id}/view`

The smoke fixture confirms that the viewer page displays:

- `Sugestii detectate`
- `Linii posibil afectate`
- `Diagnostic local · read-only`
- the raw Markdown link
- safely rendered report content

## Policy

This milestone is smoke fixture and validation only.

It does not:

- edit reports
- accept suggestions
- reject suggestions
- auto-correct formulas
- add Formula OCR
- rewrite OCR text
- rewrite pages
- rewrite course content
- change Study
- change Progress
- build packages
- create ZIP files
- deliver files
- distribute files

The fixture is temporary and local-only. It is created outside the repository during the check and is not committed.

The existing diagnostic generation policy remains unchanged: `Generate` / `Regenerează` report creation remains gated by `VOILA_ENABLE_OCR_MATH_REPORT_HOOK=1`.

## Validation

Run:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File scripts/dev/check-v0.7.4-owner-local-ocr-math-report-smoke-fixture-no-build-no-distribution.ps1
```

Expected policy: no build, no ZIP, no delivery, no distribution.
