# v0.7.0 OCR Math Diagnostic Report

Scope: owner-local only. No build, no ZIP, no delivery, no public upload, no distribution.

This milestone adds a diagnostic report layer on top of the OCR math normalizer engine.

## What it creates

For a generated PDF output folder, it writes:

- `ocr_math_report.json`
- `ocr_math_report.md`

inside:

```text
data/output/<PDF stem>/
```

## What it does not do

It does not modify:
- `pages.md`
- `course.md`
- `course.cleaned.md`
- quiz files
- Study behavior
- Progress behavior
- public UI
- release/package/delivery assets

## Purpose

The report shows:
- how many math OCR suggestions were found
- which rules triggered
- which lines are math / mixed / text
- which lines should be considered risky for formula fallback

This keeps math OCR improvement measurable before any automatic integration into Generate, Study, or UI.
