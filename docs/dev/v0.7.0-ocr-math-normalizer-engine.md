# v0.7.0 OCR Math Normalizer Engine

Scope: owner-local only. No build, no ZIP, no delivery, no public upload, no distribution.

This milestone introduces a deterministic OCR math normalizer engine separate from LanguageTool and Monaco UI.

## Why

The OCR math baseline found many math-specific issues:
- x0 / xo should often be x₀
- `V C R` / `V CR` should often be `V ⊂ ℝ`
- `x0 € R` should often be `x₀ ∈ ℝ`
- `+oo` / `-oo` should often be `+∞` / `−∞`
- math-heavy lines should be treated differently from normal Romanian prose

LanguageTool remains useful for Romanian spelling, but it is not a math OCR engine.

## Policy

The engine is deterministic and conservative:
- classify each line as text / mixed / math
- apply high-confidence math rules only on math or mixed lines
- keep suggestions in structured JSON
- do not replace formula images/crops
- do not integrate with public UI or delivery flow in this milestone

## Next integration step

Use this engine as a backend preprocessing/diagnostic layer before course/study generation, while preserving source formula screenshots for math-heavy lines.
