## Purpose

This milestone verifies that Voila starts locally cleanly after the final audit freeze, without testing write-capable product flows.

This is a runtime-readiness smoke milestone only. It is not a feature milestone, release milestone, tester package milestone, or distribution milestone.

## Explicit policy

Allowed:

- owner-local server start
- `GET /health`
- `GET /`
- existing read-only `GET` route checks, when available
- manual local browser open
- startup/log observation
- local smoke result reporting

Forbidden:

- build
- ZIP creation
- delivery
- distribution
- upload
- generate/regenerate
- save
- delete
- reset
- OCR rewrite
- course rewrite
- Study/Progress mutation
- feature changes
- behavior changes
- public UI expansion

## Smoke scope

The smoke script performs only HTTP `GET` requests against the local owner runtime.

Required checks:

- local runtime becomes reachable
- `GET /health` returns success
- `GET /` returns success

Optional read-only checks:

- `GET /quick-tools`, if route exists
- `GET /exam-prep`, if route exists

Optional routes are reported but do not fail the milestone when missing. This keeps the milestone focused on runtime readiness, not public UI expansion or route inventory changes.

## Command

Run from the repository root or from any PowerShell location:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File scripts/dev/check-owner-local-runtime-readiness-smoke-no-build-no-distribution.ps1
```

Optional manual browser open after checks:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File scripts/dev/check-owner-local-runtime-readiness-smoke-no-build-no-distribution.ps1 -OpenBrowser
```
