# v0.7.16 owner-local manual browser read-only smoke — no build / no distribution

## Baseline

Previous milestone: `v0.7.15-owner-local-runtime-readiness-smoke-no-build-no-distribution`.

Protected main baseline before this milestone:

```text
FINAL_MAIN_HEAD=30e556f
VOILA_V0_7_15_OWNER_LOCAL_RUNTIME_READINESS_SMOKE_CHECK=PASS
```

## Purpose

This milestone verifies that the local Voila runtime can be opened manually in a browser after the v0.7.15 runtime readiness smoke, while staying strictly read-only.

This is not a feature milestone, not a release milestone, not a package milestone, and not a distribution milestone.

## Policy

Allowed:

- owner-local server start
- `GET /health`
- homepage GET/read-only check
- existing read-only route checks
- manual browser open on local routes
- manual visual inspection of already-existing pages

Forbidden:

- no build
- no ZIP
- no delivery
- no distribution
- no upload
- no generate/regenerate
- no save
- no delete
- no reset
- no OCR/course/Study/Progress writes
- no feature changes
- no behavior changes
- no public UI expansion

## Manual browser read-only scope

The manual browser smoke may open only local GET/read-only URLs such as:

```text
http://127.0.0.1:8787/health
http://127.0.0.1:8787/
http://127.0.0.1:8787/quick-tools
http://127.0.0.1:8787/exam-prep
```

The owner must not use browser controls or page actions that upload, generate, regenerate, save, delete, reset, mutate progress, or rewrite any OCR/course/study data.
