# v0.3.69 — Crop Editor false figure rejection

## Summary

This milestone adds a manual safe-delete workflow for false figures detected by the hybrid figure extractor.

The Crop Editor now supports marking a detected figure as a false figure without physically deleting crop image files.

## Behavior

- Crop Editor cards include a `Mark as false figure` action.
- Rejected figures are persisted in `figures_manifest.hybrid.json`.
- Rejected figures use:
  - `status: "rejected"`
  - `reject_reason: "false_figure"`
- Rejected figures can be restored.
- Missing `status` is treated as accepted for backward compatibility.
- Hybrid gallery hides rejected figures from normal viewing.
- Crop image files are not physically deleted.

## Smoke checklist

- False figure appears before rejection.
- Clicking `Mark as false figure` hides it from normal gallery.
- Rejected state persists after browser refresh.
- `Show rejected` displays rejected cards in Crop Editor.
- `Restore` makes the figure visible again.
- Real figures remain unaffected.
- Existing manifests without `status` still work.
- No crop PNG files are physically deleted.

## Known limitation

False figure rejection is manual and user-controlled. This milestone does not change OCR, PDF processing, or automatic figure detection.

## Rollback note

Rejected figures can be restored from the Crop Editor. The original crop images remain on disk.
