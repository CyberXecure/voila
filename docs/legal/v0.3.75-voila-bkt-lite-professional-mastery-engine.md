## Summary

This milestone improves the Study mastery engine into a more professional, local, interpretable BKT-lite engine.

## Implemented changes

- Added question-type-specific BKT parameter profiles.
- Preserved the existing local Bayesian Knowledge Tracing update model.
- Added question-type difficulty metadata for:
  - definition
  - example
  - purpose
  - components
  - requirement
  - technical_fact
  - condition
  - cause_effect
  - comparison
  - process
  - numeric_check
  - visual_interpretation
- Added BKT metadata per answer attempt.
- Added per-concept metadata:
  - recent_misses
  - last_correct
  - last_incorrect
  - last_question_type
  - question_type_stats
- Improved next-question selection:
  - prioritizes low mastery
  - considers question type coverage
  - considers recent misses
  - considers question difficulty
  - avoids immediate same-concept repetition when alternatives exist
- Added interpretable recommendation reason in Study and Review UI.
- Localized recommendation reason label and values for Romanian/English.
- Kept state synchronization from v0.3.73.
