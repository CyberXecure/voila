# Voila! Tester Feedback Questions

## Goal

Collect practical feedback before building large new features.

## Product clarity

1. Is it clear what Voila! does?
2. Can you explain it in one sentence after seeing the README or tester package?
3. Is PDF-to-course / PDF-to-training a clear product idea?
4. What part is confusing?

## Use cases

Which use case feels most useful?

- course PDFs
- study materials
- manuals
- internal procedures
- onboarding documents
- technical documentation
- training content
- OCR review before content generation

## Target users

Who would benefit most?

- students
- teachers
- trainers
- self-learners
- companies
- HR / onboarding teams
- technical teams
- operations teams
- compliance teams
- documentation teams

## Output usefulness

Which output is most useful?

- course outline
- normalized outline
- cleaned course content
- glossary
- quiz
- flashcards
- OCR review artifacts

## Test document

What type of PDF did you test?

- course
- manual
- procedure
- onboarding
- technical documentation
- training document
- scanned document
- other

Approximate size:

- number of pages
- clean PDF or scanned PDF
- simple layout or complex layout

## Result quality

1. Did Voila! understand the document structure?
2. Was the generated outline useful?
3. Was the cleaned course content useful?
4. Were glossary terms useful?
5. Were quiz questions useful?
6. Were flashcards useful?
7. Was OCR review useful?
8. What output was poor or confusing?

## Practical value

1. Would this save time in a real workflow?
2. Would you use it again?
3. Would you recommend it to someone else?
4. What would make it more useful?
5. What would make it easier to test?

## Language packs

Current RC language focus:

- English
- Romanian

Future planned candidates:

- German
- Spanish

- Portuguese

Questions:

1. Are English and Romanian enough for the current stage?
2. Which language should be prioritized next?
3. Would you help review translation quality?

## Monetization

1. Would you support this project financially if it becomes useful to you?
2. What would make a paid supporter package valuable?
3. Would you prefer:
   - one-time supporter package
   - monthly sponsorship
   - paid Pro package later
   - commercial license later
4. What should stay free in the public beta?

## Final feedback

1. What is the biggest problem with Voila! right now?
2. What is the strongest part?
3. What should be improved before wider testing?
4. What should not be built yet?
