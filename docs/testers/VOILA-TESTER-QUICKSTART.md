# Voila! Tester Quickstart

## Purpose

This quickstart describes the intended experience for non-technical Voila! testers.

The future Windows tester package should be simple:

Download.
Extract.
Double-click.
Test with a PDF.
Send feedback.

## Recommended tester flow

1. Download the Voila! Windows tester package ZIP.
2. Right-click the ZIP and choose Extract All.
3. Open the extracted folder.
4. Double-click START-VOILA.bat.
5. Wait for the browser to open.
6. Open Voila! locally.
7. Test with a small PDF.
8. Review the generated outputs.
9. Stop Voila! using STOP-VOILA.bat.
10. Send feedback.

## What to test

Please test whether Voila! can help turn dense PDF documents into learning or training materials.

Good first test documents:

- course PDF
- training PDF
- onboarding document
- small internal manual
- simple procedure
- technical documentation
- small scanned PDF for OCR review

Recommended size for first test:

- 5-20 pages for clean PDFs
- 2-5 pages for scanned PDFs

## Outputs to review

Check which outputs are useful:

- course outline
- normalized outline
- cleaned course content
- glossary
- quiz
- flashcards
- OCR review artifacts

## What feedback matters most

The most useful feedback:

- Is the idea clear?
- Is the tool useful?
- Who would use it?
- Which use case is strongest?
- Which output is most valuable?
- What failed?
- What was confusing?
- Would this save time?
- Would this be useful for courses, manuals, procedures, onboarding, training or technical documentation?
- Would a paid supporter package make sense later?

## What not to test yet

Avoid testing with:

- confidential documents
- personal data
- legal records
- medical records
- financial records
- safety-critical content
- very large PDFs
- complex tables or forms
- image-heavy PDFs
- scanned books with hundreds of pages

## Important note

Voila! is still in public beta.

Generated content must be reviewed by a human before being used seriously.
