# Voila Beta EULA Commercial Readiness Notes

## Purpose

Connect the beta EULA draft to Voila commercial readiness.

This document is planning-only and does not enable payment, activation, Supporter builds, or Pro builds.

---

## Current direction

Voila is moving toward:

```text
proprietary product
private development
controlled public releases
clear beta terms
future Supporter / Pro licensing
```

---

## What the beta EULA supports

The beta EULA draft helps with:

```text
public beta clarity
tester demo restrictions
no-redistribution position
no-resale position
generated content disclaimer
user document responsibility
third-party component notice reference
future package legal structure
```

---

## What the beta EULA does not solve

It does not yet solve:

```text
final legal approval
Supporter purchase terms
Pro commercial license terms
refund policy
privacy policy
activation terms
tax/payment terms
third-party license audit completion
code signing
installer EULA acceptance
```

---

## Before Supporter package

Recommended minimum:

```text
[ ] beta EULA reviewed
[ ] Supporter rights separately defined
[ ] Supporter page limit / feature set defined
[ ] no-redistribution wording confirmed
[ ] personal/internal use wording confirmed
[ ] package legal files included
[ ] third-party notices reviewed enough for distribution
```

---

## Before Pro package

Recommended minimum:

```text
[ ] Pro EULA or Pro addendum drafted
[ ] commercial/internal use rights defined
[ ] support expectations defined
[ ] third-party license audit completed
[ ] payment/refund terms prepared
[ ] official download channel chosen
[ ] package legal folder included
[ ] release type clearly labeled
```

---

v0.3.11-voila-package-legal-files-integration-plan

## Recommendation

Use the beta EULA draft as a foundation, but do not treat it as final commercial terms until reviewed.

For paid distribution, create a separate Supporter/Pro license addendum or paid EULA section.
