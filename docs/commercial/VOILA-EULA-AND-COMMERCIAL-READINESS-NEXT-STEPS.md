# Voila EULA and Commercial Readiness Next Steps

## Purpose

Connect the EULA planning work to Voila's commercial readiness path.

This document is planning-only and does not implement payment, activation, packaging, or repository visibility changes.

## Current product direction

Voila should continue as:

```text
proprietary product
private development repository
controlled public releases
clear beta terms
future Supporter / Pro licensing
```

## Why EULA comes before monetization

Before asking users to pay or distributing wider packages, Voila should have:

```text
clear usage rights
clear redistribution restrictions
clear beta limitations
clear generated-content disclaimer
clear third-party notices
clear package type labels
clear Supporter / Pro distinction
```

## Recommended sequence

### Step 1 — EULA plan

Current milestone:

```text

```

Goal:

### Step 4 — Package legal integration plan

Possible next milestone:

```text
v0.3.11-voila-package-legal-files-integration-plan
```

```text
v0.4.1-voila-supporter-package-plan
```


## Commercial readiness checklist

Before paid distribution:

```text
[ ] EULA draft reviewed
[ ] beta terms reviewed
[ ] third-party notices completed
[ ] release type labels finalized
[ ] package legal file structure finalized
[ ] public download wording is clear
[ ] Supporter rights defined
[ ] Pro rights defined
[ ] refund/payment approach defined
[ ] support/contact approach defined
[ ] repository privacy decision completed
[ ] official download channel chosen
[ ] SHA256 process documented
[ ] package smoke test documented
```

## What remains unresolved

```text
final legal review
third-party license audit
privacy policy
payment terms
refund policy
activation approach
Supporter vs Pro page limits
commercial use boundaries
installer/EULA acceptance UI
code signing
```

## Recommendation

Do not launch a paid Supporter or Pro package until:

```text
EULA exists
third-party notices are reviewed
package legal files are included
public download wording is clear
release type is clear
support/contact path is clear
```
