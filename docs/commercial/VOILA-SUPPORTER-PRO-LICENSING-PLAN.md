# Voila Supporter and Pro Licensing Plan

## Purpose

Plan the future commercial packaging direction for Voila.

This is not an implementation milestone. It does not add payment, activation, licensing code, or runtime restrictions.

---

## Recommended tiers

### Free beta / tester demo

Purpose:

```text
collect feedback
validate usability
show product value
avoid overpromising
```

Possible limits:

```text
small page count
limited support
non-commercial use
beta terms apply
```

### Supporter

Purpose:

```text
early monetization
support development
reward early adopters
validate willingness to pay
```

Possible benefits:

```text
higher page limit
early builds
priority feedback consideration
supporter badge or acknowledgment
personal use license
```

### Pro

Purpose:

```text
commercial use
professional users
training/documentation workflows
higher-value features
```

Possible benefits:

```text
larger page limits
commercial/internal use rights
export improvements
batch processing later
better OCR tools
priority support later
```

---

## Licensing principles

```text
keep terms simple
avoid unlimited promises early
separate personal/supporter use from commercial/pro use
keep beta and production terms separate
require explicit permission for redistribution
keep source code private
```

---

## What to avoid early

Avoid too early:

```text
complex subscriptions
enterprise contracts
heavy license server infrastructure
overly broad lifetime licenses
promising unlimited OCR/PDF processing
promising production accuracy
```

---

## Practical first monetization step

Recommended first paid experiment:

```text
Voila Supporter Package
one-time payment
personal/internal use
higher page limit than demo
early access builds
clear no-redistribution terms
```

---

## Later Pro direction

Pro can be introduced after:

```text
public beta feedback is positive
runtime package is stable
third-party license review is completed
EULA is ready
support process exists
basic licensing/activation approach is chosen
```

---

## Open questions

```text
Should Supporter be one-time or annual?
Should Pro be one-time, annual, or subscription?
Should activation be offline or online?
Should commercial use require Pro?
What page limit is fair per tier?

```
