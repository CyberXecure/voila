# Voila Windows ZIP Candidate START/STOP Smoke Commercial Gate

## Purpose

Define the commercial/release gate around START/STOP smoke validation.

This is guidance only.

---

## Gate before sharing candidate

Required:

```text
[ ] ZIP candidate build result documented
[ ] SHA256 recorded
[ ] extract validation passed
[ ] START/STOP smoke plan completed
[ ] START/STOP smoke result PASS or documented CONDITIONAL
[ ] README reviewed
[ ] release notes reviewed
[ ] legal folder verified
```

---

## Gate before GitHub release

Required later:

```text
[ ] smoke result documented
[ ] known warnings documented
[ ] support/feedback path defined
[ ] release notes finalized
[ ] SHA256 asset ready
[ ] public wording reviewed
```

---

## Commercial boundary

START/STOP smoke validation does not mean commercial readiness.

Before Supporter/Pro distribution:

```text
[ ] paid terms ready
[ ] EULA reviewed for paid use
[ ] third-party notices complete enough for distribution
[ ] payment/refund wording reviewed
[ ] license/activation plan decided
```

---

## Gate statuses

Use:

```text
Pass
Conditional
Fail
Not reviewed
```
