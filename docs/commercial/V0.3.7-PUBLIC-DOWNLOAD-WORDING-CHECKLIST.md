## Purpose

Clarify public download wording for Voila releases.

The public-facing message should not imply that every latest GitHub release is the correct runtime package for users.

---

## Problem

The phrase:

```text
Download the latest release.
```

can be misleading because Voila releases may include:

```text
public beta runtime packages
tester demo builds
language-pack release candidates
documentation-only updates
limited builds
internal builds
future Supporter / Pro builds
```

---

## Recommended wording

Use:

```text
Download the public beta runtime package identified in the release notes.
```

Avoid:

```text
Download the latest release.
```

---

## Release type definitions

### Public Beta Runtime Package

```text
For public evaluation and feedback.
Should include a runnable package.
Release notes should state that it is the recommended public runtime package.
```

### Tester Demo Build

```text
For selected testers.
May include page-count limits or other demo restrictions.
Not necessarily the same as the public runtime package.
```

### Language Pack Release Candidate

```text
For language-pack validation or packaging readiness.
May not be intended as a general runtime download.
```

### Documentation-only Update

```text
Updates README, public docs, screenshots, legal/commercial planning, or release wording.
Does not change runtime behavior.
```
